1.1 WHAT IS DOCKER?
Docker is a platform that enables developers to package applications into containers—standardized executable components that combine application source code with operating system (OS) libraries and dependencies required to run that code in any environment.

The Problem Docker Solves:
Before Docker: "It works on my machine!"
After Docker: "It works everywhere!"

Key Benefits:
✓ Consistency across development, testing, and production
✓ Isolation between applications
✓ Portability across any system with Docker
✓ Efficiency (lightweight compared to VMs)
✓ Scalability (easy to scale horizontally)
✓ Version control for application environments
✓ Faster deployment and onboarding

Docker Image
Read-only template with instructions for creating a container

Built in layers (each instruction adds a layer)

Stored in a registry (Docker Hub, private registry)

Immutable (once built, doesn't change)

Docker Container
Runnable instance of an image

Isolated from host and other containers

Has its own filesystem, network, and process tree

Can be started, stopped, moved, and deleted

Dockerfile
Text file with instructions to build an image

Automates image creation

Version controllable like source code

Docker Registry
Storage and distribution system for images

Docker Hub (public), Amazon ECR, Google Container Registry, etc.

Can host private registries

Docker Compose
Tool for defining multi-container applications

Uses YAML files for configuration

One command to start entire application stack
