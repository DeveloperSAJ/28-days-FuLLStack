# ⌨️ (0:00) Introduction

MySQL is:

* Open-source Relational Database Management System (RDBMS)
* Uses SQL (Structured Query Language)
* Stores data in tables
* Follows ACID principles (when using InnoDB)

Used in:

* Web applications
* Enterprise systems
* E-commerce
* SaaS products

# ⌨️ (2:36) What is a Database?

A **database** is an organized collection of data.

Instead of storing data in:

* Excel files
* JSON files
* Text files

We use databases because they provide:

* Structured storage
* Fast querying
* Data integrity
* Security
* Relationships between data

---

## Types of Databases

1. Relational (SQL) → MySQL
2. NoSQL → MongoDB, Firebase
3. In-memory → Redis

# ⌨️ (23:10) Tables & Keys

## 🔹 Tables

Tables consist of:

* Rows (records)
* Columns (fields)

Example:

| id | name | email |
| -- | ---- | ----- |

---

## 🔹 Primary Key

* Uniquely identifies each row
* Cannot be NULL
* Must be unique

```sql
PRIMARY KEY (id)
```

---

## 🔹 Foreign Key

* Links one table to another
* Creates relationships

```sql
FOREIGN KEY (user_id) REFERENCES users(id)
```

---

## 🔹 Composite Key

Multiple columns combined as primary key.

# ⌨️ (43:31) SQL Basics

SQL has 4 major categories:

1. DDL – Data Definition Language

   * CREATE
   * DROP
   * ALTER

2. DML – Data Manipulation Language

   * INSERT
   * UPDATE
   * DELETE

3. DQL – Data Query Language

   * SELECT

4. DCL – Data Control Language

   * GRANT
   * REVOKE

Combine results of multiple SELECT statements.


Rules:

* Compatible data types

# ⌨️ (3:01:36) Joins

## INNER JOIN

```sql
SELECT employees.first_name, branch.branch_name
FROM employees
JOIN branch
ON employees.branch_id = branch.branch_id;
```

## LEFT JOIN

Returns all left table rows.

## RIGHT JOIN

Returns all right table rows.

# ⌨️ (3:11:49) Nested Queries (Subqueries)

Query inside query.

Example:

```sql

FROM employees

    SELECT AVG(salary) FROM employees
);
```

# ⌨️ (3:21:52) ON DELETE

Used with foreign keys.

```sql
FOREIGN KEY (branch_id)

ON DELETE CASCADE;
```

Options:

* CASCADE
* SET NULL
* RESTRICT
* NO ACTION

# ⌨️ (3:30:05) Triggers

Automatically execute SQL when event happens.

Example:

```sql
CREATE TRIGGER before_insert_employee
BEFORE INSERT ON employees
FOR EACH ROW
SET NEW.salary = IFNULL(NEW.salary, 30000);
```

# ⌨️ (3:55:53) Designing an ER Diagram

Steps:

1. Identify entities (Employee, Branch)
2. Identify attributes
3. Define primary keys
4. Define relationships
5. Define cardinality (1:1, 1:M, M:N)

---

# ⌨️ (4:08:34) Converting ER Diagrams to Schemas

Rules:

1. Entity → Table
2. Attribute → Column
3. Primary key → PRIMARY KEY
4. 1:M → Foreign key in "many" table
5. M:N → Create junction table

Example M:N:

```sql
CREATE TABLE works_with (
    emp_id INT,
    client_id INT,
    PRIMARY KEY (emp_id, client_id),
    FOREIGN KEY (emp_id) REFERENCES employees(emp_id),
    FOREIGN KEY (client_id) REFERENCES clients(client_id)
);
```
