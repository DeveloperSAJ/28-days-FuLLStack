
# ⌨️ (0:00) Introduction

Firebase is a **Backend-as-a-Service (BaaS)** platform by Google.

Instead of building:

Frontend → Backend → Database → Auth → Hosting → Server

You directly use:

Frontend → Firebase Services

Firebase handles:

* Infrastructure
* Scaling
* Security
* Server maintenance

It is ideal for:

* MVPs
* Startups
* Mobile apps
* Real-time apps

---

# ⌨️ (What is Firebase?)

Firebase is NOT just a database.

It includes:

1. Firestore (database)
2. Authentication
3. Cloud Functions
4. Hosting
5. Cloud Storage
6. Analytics
7. Push notifications

It’s a complete backend ecosystem.

# ⌨️ (Firebase Architecture Overview)

* 🔹 Firestore → Store data

Everything connects inside one project.

# ⌨️ Firestore Data Structure

Firestore structure:

Collection → Document → Fields → Subcollection

Example:

```
users (collection)
   └── user123 (document)
          ├── name: "Ali"
          ├── age: 22
          └── orders (subcollection)
                └── order001
```

Important rules:

* Collections contain documents
* Documents contain fields
* Documents can contain subcollections
* No tables
* No joins

# ⌨️ Installing Firebase (Web Example)

```bash
npm install firebase
```

Initialize:

```javascript
import { initializeApp } from "firebase/app";
import { getFirestore } from "firebase/firestore";

const app = initializeApp(firebaseConfig);
const db = getFirestore(app);
```

Explanation:

* initializeApp connects your app to Firebase
* getFirestore gives access to the database

---

# ⌨️ Adding Data to Firestore

Add a document:

```javascript
import { collection, addDoc } from "firebase/firestore";

await addDoc(collection(db, "users"), {
  name: "Ali",
  age: 22
});
```

Explanation:

* collection(db, "users") → reference to collection
* addDoc → adds document with auto ID

---

# ⌨️ Reading Data

```javascript
import { getDocs } from "firebase/firestore";

const snapshot = await getDocs(collection(db, "users"));

snapshot.forEach((doc) => {
  console.log(doc.id, doc.data());
});
```

Explanation:

* getDocs retrieves all documents
* doc.data() returns fields

---

# ⌨️ Querying Data

```javascript
import { query, where } from "firebase/firestore";

const q = query(
  collection(db, "users"),
  where("age", ">", 20)
);
```

Important:

* No joins
* Queries must match indexes
* You can combine filters (with limits)

---

# ⌨️ Real-Time Listeners (Firebase Superpower)

```javascript
import { onSnapshot } from "firebase/firestore";

onSnapshot(collection(db, "users"), (snapshot) => {
  snapshot.forEach((doc) => {
    console.log(doc.data());
  });
});
```

Explanation:

* Listens to changes
* Updates UI automatically
* No manual refresh

Used in:

* Chat apps
* Dashboards
* Notifications

---

# ⌨️ Updating & Deleting Data

Update:

```javascript
import { updateDoc, doc } from "firebase/firestore";

await updateDoc(doc(db, "users", "userId"), {
  age: 23
});
```

Delete:

```javascript
import { deleteDoc } from "firebase/firestore";

await deleteDoc(doc(db, "users", "userId"));
```

* Sessions

# ⌨️ Security Rules (Very Important)

Security is NOT automatic.

Example rule:

```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /users/{userId} {
      allow read, write: if request.auth != null;
    }
  }
}
```

Explanation:

* Only authenticated users can access
* You can restrict by user ID
* You can implement role-based access

Never leave production in test mode.

# ⌨️ Firebase Storage

Stores:

* Images
* Videos
* PDFs

Example:


# ⌨️ Firestore vs Realtime Database

Older database:
👉 Firebase Realtime Database

Differences:

| Firestore     | Realtime DB |
| ------------- | ----------- |
| Structured    | Big JSON    |
| Scales better | Limited     |
| Modern        | Legacy      |

Use Firestore for new apps.

# ⌨️ Data Modeling in Firestore

Important concepts:

* Avoid deep nesting
* Use subcollections for scalable data
* Duplicate data for faster reads
* Think in query patterns

Example:
Instead of joins:
Store username inside post document.
