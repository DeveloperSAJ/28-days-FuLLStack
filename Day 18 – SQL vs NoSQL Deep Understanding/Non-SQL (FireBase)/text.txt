
# 1️⃣ What Is MySQL?

MySQL is:

* Open-source relational database
* Uses Structured Query Language (SQL)
* Based on table structure (rows + columns)
* Follows ACID properties

Used in:

* Web apps
* E-commerce
* Banking systems
* SaaS platforms

---

# 2️⃣ MySQL Architecture (Important for Interviews)

### Core Components:

1. **Server**

   * Handles queries
   * Manages connections
   * Optimizes execution

2. **Storage Engine**

   * InnoDB (default)
   * MyISAM (older)

### 🔥 InnoDB (Most Important)

* Supports transactions
* Row-level locking
* Foreign keys
* Crash recovery

Interview question:

> Why InnoDB over MyISAM?

Answer:

* Transactions
* Better concurrency
* Foreign key support

# 3️⃣ Database & Table Commands (DDL)

DDL = Data Definition Language

## 🔹 Create Database

```sql
CREATE DATABASE company;
```

## 🔹 Show Databases

```sql
SHOW DATABASES;
```

## 🔹 Use Database

```sql
USE company;
```

## 🔹 Drop Database

```sql
DROP DATABASE company;
```

---

## 🔹 Create Table

```sql
CREATE TABLE users (
    id INT PRIMARY KEY AUTO_INCREMENT,
    name VARCHAR(100) NOT NULL,
    email VARCHAR(100) UNIQUE,
    age INT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

Concepts used:

* PRIMARY KEY
* AUTO_INCREMENT
* NOT NULL
* UNIQUE
* DEFAULT

---

## 🔹 Alter Table

```sql
ALTER TABLE users ADD phone VARCHAR(20);
ALTER TABLE users MODIFY name VARCHAR(200);
ALTER TABLE users DROP phone;
```

---

## 🔹 Drop Table

```sql
DROP TABLE users;
```

---

# 4️⃣ Data Manipulation Commands (DML)

## 🔹 Insert

```sql
INSERT INTO users (name, email, age)
VALUES ('Ali', 'ali@gmail.com', 22);
```

Insert multiple:

```sql
INSERT INTO users (name, email, age)
VALUES 
('Ahmed', 'ahmed@gmail.com', 25),
('Sara', 'sara@gmail.com', 21);
```

---

## 🔹 Select (MOST IMPORTANT)

```sql
SELECT * FROM users;
```

Specific columns:

```sql
SELECT name, email FROM users;
```

---

### 🔥 WHERE

```sql
SELECT * FROM users WHERE age > 20;
```

Operators:

* =
* !=
* >
* <
* BETWEEN
* IN
* LIKE
* AND / OR

Example:

```sql
SELECT * FROM users 
WHERE age BETWEEN 20 AND 25;
```

---

### 🔥 ORDER BY

```sql
SELECT * FROM users ORDER BY age DESC;
```

---

### 🔥 LIMIT

```sql
SELECT * FROM users LIMIT 5;
```

Pagination:

```sql
SELECT * FROM users LIMIT 5 OFFSET 10;
```

---

## 🔹 Update

```sql
UPDATE users
SET age = 23
WHERE id = 1;
```

⚠️ Always use WHERE.

---

## 🔹 Delete

```sql
DELETE FROM users WHERE id = 1;
```

Delete all:

```sql
DELETE FROM users;
```

# 5️⃣ Constraints

* PRIMARY KEY
* FOREIGN KEY
* UNIQUE
* NOT NULL
* CHECK
* DEFAULT

Example with Foreign Key:

```sql
CREATE TABLE orders (
    id INT PRIMARY KEY AUTO_INCREMENT,
    user_id INT,
    FOREIGN KEY (user_id) REFERENCES users(id)
);
```

# 6️⃣ Joins (Very Important for Interviews)

Assume:

* users
* orders

---

## 🔹 INNER JOIN

```sql
SELECT users.name, orders.id
FROM users
INNER JOIN orders
ON users.id = orders.user_id;
```

---

## 🔹 LEFT JOIN

Returns all users even if no orders.

```sql
SELECT users.name, orders.id
FROM users
LEFT JOIN orders
ON users.id = orders.user_id;
```

---

## 🔹 RIGHT JOIN

Opposite of LEFT JOIN.

---

# 7️⃣ Aggregation Functions

* COUNT()
* SUM()
* AVG()
* MIN()
* MAX()

Example:

```sql
SELECT COUNT(*) FROM users;
```

---

## 🔹 GROUP BY

```sql
SELECT age, COUNT(*) 
FROM users
GROUP BY age;
```

---

## 🔹 HAVING

```sql
SELECT age, COUNT(*) 
FROM users
GROUP BY age
HAVING COUNT(*) > 2;
```

Difference:

* WHERE filters rows
* HAVING filters grouped results

# 8️⃣ Indexing (Performance Topic)

Create index:

```sql
CREATE INDEX idx_email ON users(email);
```

Why use index?

* Faster search
* Faster WHERE conditions
* Faster JOIN

But:

* Slower INSERT/UPDATE
* Takes memory

---

# 9️⃣ Transactions (ACID)

```sql
START TRANSACTION;

UPDATE users SET balance = balance - 100 WHERE id = 1;
UPDATE users SET balance = balance + 100 WHERE id = 2;

COMMIT;
```

Rollback:

```sql
ROLLBACK;
```

Used in:

* Banking
* Payments
* Critical operations

---

# 🔟 Views

```sql
CREATE VIEW active_users AS
SELECT * FROM users WHERE age > 18;
```

Acts like virtual table.

# 1️⃣2️⃣ Triggers

Automatically runs after/before events.

```sql
CREATE TRIGGER before_insert_user
BEFORE INSERT ON users
FOR EACH ROW
SET NEW.created_at = NOW();
```
