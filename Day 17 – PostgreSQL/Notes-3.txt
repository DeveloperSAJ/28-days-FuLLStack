

## ⌨️ What is a Database

A **database** is an organized collection of data stored electronically.

Instead of:

* Excel files
* Paper records
* Random JSON files

A database:

* Stores data in structured format
* Allows fast querying
* Supports security & multi-user access
* Ensesses data integrity

---

## ⌨️ What is SQL And Relational Database

### SQL

SQL = **Structured Query Language**
Used to:

* Create data
* Read data
* Update data
* Delete data

(CRUD operations)

### Relational Database

Data is stored in **tables**
Tables relate to each other using **keys**

Example:

```
users table
orders table
```

Orders reference users.

---

## ⌨️ What is PostgreSQL (Postgres)

**PostgreSQL**

* Open-source relational database
* ACID compliant
* Very powerful
* Supports JSON, indexing, extensions
* Used in production systems

# PART 2 — Installation & Setup

---

## ⌨️ PostgreSQL Installation (Mac OS)

1. Download installer
2. Install with default settings
3. Set password for `postgres` user
4. Use:

```bash
psql -U postgres
```

---

## ⌨️ PostgreSQL Installation (Windows)

1. Download Windows installer
2. Install PostgreSQL server
3. Install pgAdmin
4. Set password
5. Open SQL Shell (psql)

---

## ⌨️ GUI Clients vs Terminal/CMD Clients

### GUI (pgAdmin)

* Visual interface
* Easy for beginners

### Terminal (psql)

* Faster
* More powerful
* Used by professionals

---

## ⌨️ Setup PSQL (Mac & Windows)

Add PostgreSQL to system PATH.

Test:

```bash
psql --version
```

Login:

```bash
psql -U postgres
```

# PART 4 — Tables

---

## ⌨️ How To Create Tables

```sql
CREATE TABLE person (
    id INT,
    name TEXT
);
```

---

## ⌨️ Creating Tables Without Constraints

Basic structure:

```sql
CREATE TABLE person (
    id INT,
    first_name TEXT,
    gender TEXT
);
```

No validation rules.

---

## ⌨️ Creating Tables with Constraints

```sql
CREATE TABLE person (
    id SERIAL PRIMARY KEY,
    email TEXT UNIQUE NOT NULL,
    gender TEXT CHECK (gender IN ('Male','Female'))
);
```

Constraints enforce rules.

# PART 6 — Querying Data

---

## ⌨️ Select From

```sql
SELECT * FROM person;
```

---

## ⌨️ Order By

```sql
SELECT * FROM person
ORDER BY first_name ASC;
```

---

## ⌨️ Distinct

```sql
SELECT DISTINCT gender FROM person;
```

---

## ⌨️ Where Clause and AND

```sql
SELECT * FROM person
WHERE gender = 'Male'
AND country = 'USA';
```

---

## ⌨️ Comparison Operators

* `=`
* `!=`
* `>`
* `<`
* `>=`
* `<=`

---

## ⌨️ Limit, Offset & Fetch

```sql
SELECT * FROM person
LIMIT 10 OFFSET 5;
```

Pagination.

---

## ⌨️ IN

```sql
WHERE country IN ('USA','Canada');
```

---

## ⌨️ Between

```sql

```

---

## ⌨️ Like And iLike

```sql

```

`ILIKE` = case insensitive.

# PART 11 — Update & Delete

---

## ⌨️ Delete Records

```sql
DELETE FROM person WHERE id = 5;
```

---

## ⌨️ Update Records

```sql
UPDATE person
SET first_name = 'Ahmed'
WHERE id = 1;
```

---

## ⌨️ On Conflict Do Nothing

```sql
ON CONFLICT (email) DO NOTHING;
```

---

## ⌨️ Upsert

```sql
ON CONFLICT (email)
DO UPDATE SET first_name = EXCLUDED.first_name;
```

Insert or update.

# PART 12 — Relationships

---

## ⌨️ Foreign Keys

```sql
FOREIGN KEY (user_id)
REFERENCES users(id);
```

Connect tables.

---

## ⌨️ Inner Join

```sql
SELECT *
FROM orders
JOIN users
ON users.id = orders.user_id;
```

---

## ⌨️ Left Join

```sql
LEFT JOIN
```

Returns all left rows.

---

## ⌨️ Deleting With Foreign Keys

Need:

* `ON DELETE CASCADE`
  or
* delete child first
